# Shootout Full Support Plan

This document describes how to bring rpyforth from **6 of 23** classic
[Doug Bagley / Win32 Shootout](http://www.bagley.org/~doug/shootout/) Forth
programs to full coverage of the original Gforth benchmark set used in early
2000s comparisons.

It complements `FORTH2012_COVERAGE.md` (Core wordset) and
`SHOOTOUT_COVERAGE.md` (generated by `check_coverage.py`).

## Design principles

1. **Prefer ANSI-style ports** where Gforth used extensions (already done for
   `heap.fs`). Keeps benchmarks maintainable without `REQUIRE objects.fs`.
2. **Lazy heap** stays separate from the JIT hot interpreter state (see
   `rpyforth/heap.py`) so non-memory benchmarks (e.g. nestedloop) pay no cost.
3. **Add words in layers**: file I/O → exceptions → dictionary → structures.
4. **Verify with `run_shootout.py --compare gforth`** after each tier; add
   `curve/` variants when a benchmark is stable.
5. **Do not block on OO / threads / parser generator** for “full support” of
   the *algorithmic* shootout set; provide simplified ports or mark as
   `out-of-scope` with documented alternatives.

## Implementation phases

### Phase 0 — Polish existing six (1–2 days)

**Goal:** Match original shootout CLI behaviour and reporting.

| Task | Words / infra | Benchmarks unlocked |
|------|---------------|---------------------|
| Implement `ARG` (nth argv as counted string) | `ARG` | All (dynamic NUM) |
| Wire `argc @ 1- arg >number` in all `.fs` files | existing `ARGC` | All |
| Add `RECURSIVE` as immediate alias for `RECURSE` | `RECURSIVE` | ack, fibo |
| Document hard-coded NUM fallback | — | CI stability |
| Add curve copies only where missing | — | parity with runbench |

**Exit criteria:** Six programs run with optional CLI argument; gforth
comparison unchanged or improved.

---

### Phase 1 — File I/O and strings (1–2 weeks)

**Goal:** Text-processing shootout programs without Gforth-specific libraries.

| Task | Priority words | Programs |
|------|----------------|----------|
| Standard file words | `READ-FILE`, `READ-LINE`, `WRITE-FILE`, `OPEN-FILE`, `CLOSE-FILE` | sumcol, reversefile, wc, spellcheck |
| File constants | `STDIN`, `STDOUT`, `STDERR` (or fd 0/1) | sumcol, reversefile, wc |
| Memory resize | `RESIZE` | strcat |
| String copy | `CMOVE`, `CMOVE>` | strcat, wc |
| Control | `CASE`, `OF`, `ENDOF`, `ENDCASE` | wc |
| Trivial | `.\"`, `BYE` already present | hello |

**Suggested port order:** hello → sumcol → strcat → reversefile → wc

**Exit criteria:** Five new programs in `shootout/`; `run_shootout.py` green
vs gforth on result values.

---

### Phase 2 — Exceptions and dynamic state (3–5 days)

| Task | Words | Programs |
|------|-------|----------|
| Exception handling | `CATCH` (with existing `THROW`) | except |
| Modifiable definitions | `VALUE`, `TO` | random, prodcons (if ported) |
| Float print helpers | `F$` or minimal `TYPE` of float digits | random |

**Port order:** except → random (rewrite without `EVALUATE` hack)

---

### Phase 3 — Search-order dictionary (2–3 weeks)

**Goal:** Hash and spell benchmarks without full Gforth dictionary model.

| Task | Words | Programs |
|------|-------|----------|
| Minimal wordlists | `WORDLIST`, `SEARCH-WORDLIST`, `GET-CURRENT`, `SET-CURRENT` | hash, spellcheck |
| Define from parsed name | `NEXTNAME` or port helper | hash, hashcheck |
| Search order | `>ORDER`, `PREVIOUS`, `WORDLIST-ID` | hash2 |
| Name fields | `NAME>INT`, `NAME>STRING`, `LASTXT` | hash2, wordfreq |

**Port order:** hash → spellcheck → hash2 → wordfreq (wordfreq also needs Phase 4)

---

### Phase 4 — Structures and float heap extensions (2 weeks)

| Task | Words | Programs |
|------|-------|----------|
| Record layout | `STRUCT`, `END-STRUCT`, `FIELD`, `CELL%` | lists, wordfreq, moments |
| Heap allocation | `%ALLOT` / portable `ALLOCATE` wrapper | lists |
| Float array compile | `F,` | moments |
| String to float | `>FLOAT` | moments, spellcheck |
| Conditional loops | `?DO` | lists, hash, many originals |
| Sort / quantile | port in Forth or small primitive helpers | moments, wordfreq |

**Port order:** lists → moments → wordfreq (finish)

---

### Phase 5 — Matrix multiply (1 week)

| Task | Words | Programs |
|------|-------|----------|
| Vector primitive or colon `v*` | `V*` or optimized triple loop | matrix |
| Already have | `F@`, `F!`, `FLOATS`, `POSTPONE`, `CREATE`, `ALLOT` | matrix |

Prefer a **simplified matrix.fs** that avoids immediate `POSTPONE`/`[IS]` if
possible.

---

### Phase 6 — Out of scope / alternative ports (optional)

These originals depend on Gforth libraries that are poor fits for rpyforth.

| Program | Gforth dependency | Recommendation |
|---------|-------------------|----------------|
| methcall | `objects.fs` | Skip or rewrite with plain colon words + records |
| objinst | `objects.fs` | Same |
| prodcons | `tasker.fs` | Skip (no threading model) or fake with sequential code |
| regexmatch | `gray.fs` | Hand-written recursive-descent port (~200 lines) |

Document as **Tier C — not required for numeric shootout parity**.
